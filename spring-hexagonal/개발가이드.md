# 개발가이드

## 아키텍처
- 헥사고날 아키텍처
- 도메인 모델 패턴

### 계층
- Domain Layer
- Application Layer
- Adapter Layer

> 외부(Actor) -> 어댑터 -> 애플리케이션 -> 도메인

## 패키지
- domain:  핵심 비즈니스 로직과 엔티티 (순수 Java, 외부 의존 X)
- application: 유스케이스 구현 및 포트 인터페이스 정의
  - required: 외부가 구현해야 할 포트 (Repository, ExternalAPI 등의 인터페이스)
  - provided: 외부에 제공할 포트 (UseCase, Service 등의 인터페이스)
- adapter: 외부 기술과 연결하는 어댑터 구현체
  - web: REST API, Controller (Driving Adapter)
  - persistence: DB 접근 구현 (Repository Impl, JPA Entity)
  - integration: 외부 API 연동 (Payment, Email 등)
  - security: 인증/인가 구현 (Spring Security 등)
